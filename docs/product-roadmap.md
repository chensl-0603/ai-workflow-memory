# 产品路线

ai-workflow-memory 的目标是成为一个轻量但可靠的本地 AI 工作流记忆系统：自动采集 AI 协作开发上下文，沉淀为 SQLite 事实源，并同步到 Obsidian 作为长期阅读和人工补充层。

## 三条主线

### 记忆管理

目标是让 Codex / Claude 会话不会只停留在线程标题里。

已落地能力：

- Codex `session_index.jsonl` 和 Claude history / projects 采集。
- 会话摘要、标签、项目路径、来源引用写入 SQLite。
- 记忆搜索、标签筛选、摘要质量审计。
- 标题兜底、正文备份、可恢复性诊断。
- 人工摘要优先级和撤回机制。

后续方向：

- 更细的高价值记忆补摘要工作台。
- 长期无新记忆但项目仍变化时的提醒。

### 项目管理复盘

目标是把“今天发生了什么”升级为“哪些项目被推进、哪些阻塞反复出现、下一步应该做什么”。

已落地能力：

- Daily 复盘和行动收件箱。
- Goals、Decisions、Blockers、Strategy 跨项目视图。
- 项目档案导出和手动备注解析。
- 项目阶段快照事实源。
- 阶段复盘草稿，记录完成内容、验证命令、提交记录、遗留问题和下一步。
- 行动闭环证据，关联提交、测试结果、同步记录和完成来源。

后续方向：

- 周/月维度跨项目推进汇总。
- 更自动的下一阶段建议排序。

### 开发环境监测

目标是把工具缺失、env 缺失、权限异常和同步失败从偶发报错变成可追踪的项目风险。

已落地能力：

- 本地项目扫描、技术栈识别、Git 状态识别。
- Node/npm/Python/Java/Maven/Gradle、env、wrapper 文件检查。
- 项目级健康异常进入阻塞看板和行动建议。
- 健康检查历史趋势，识别反复出现的环境异常。
- 同步运行历史、同步目标文件快照和失败诊断。

后续方向：

- 更多技术栈检查，例如 pnpm、Python venv、Docker、数据库连接配置。
- 将 Codex 权限异常纳入环境监测主线。
